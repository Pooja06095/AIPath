📌 What is a Git Repository?

A Git Repository (Repo) is a folder where Git stores all the files, their history, and every change made to the project.

💻 Initialize a Repository

To start tracking a project with Git, open the project folder in Command Prompt or Git Bash and run:

git init

This creates a hidden .git folder that stores all version history.

📸 Command Preview

After running git init, Git initializes the repository and starts tracking changes.

📄 Check Repository Status

Use this command to check the current status of your repository.

git status

It shows modified files, new files, and files ready to commit.

📂 Add Files to Staging Area

Before saving changes, Git needs to know which files you want to include in the next commit.

git add .

This command adds all modified and new files to the staging area.

📝 Save Changes with Commit

A commit saves a snapshot of your project. Always write a meaningful commit message.

git commit -m "Initial Commit"

This command permanently records your staged changes in Git history.

🔄 Git Workflow

Basic Git workflow:

Working Directory → git add → Staging Area → git commit → Repository

⚠️ Common Mistakes

  • Forgetting to run git add before committing.
  • Writing unclear commit messages like "update".
  • Running Git commands outside the project folder.
  • Ignoring the output of git status.

💡 Pro Tip

Run git status before every commit. It helps you verify which files will be included.

🧠 Common Interview Question

Q. What is the difference between git add and git commit?

Answer: git add moves files to the staging area, while git commit permanently saves those changes in Git history.

💡 Did You Know?

Developers working on large projects may create hundreds of commits every week to track changes efficiently.

📋 Key Points

  • git init creates a new Git repository.
  • git status checks the repository status.
  • git add . stages all changes.
  • git commit saves changes permanently.
