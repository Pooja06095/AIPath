🌿 Git Branching

Learn how Git Branches help developers work on different features independently without affecting the main project.

Lesson 5 of 8

📌 What is a Branch?

A branch is an independent line of development in Git. It allows developers to work on new features or fix bugs without changing the main project.

💡 Why Use Branches?

🚀 Develop Features Safely
🐞 Fix Bugs Separately
👨‍💻 Team Collaboration
🔄 Merge Changes Easily

🏢 Real World Example

Imagine you are developing an E-commerce website.

  • 👨 Rahul is working on the Login page.
  • 👩 Priya is working on the Payment page.
  • 👨 Aman is fixing bugs.

If everyone works on the main branch, they may overwrite each other's work.

Therefore, each developer creates a separate branch and later merges it into the main branch.

🌿 Create a Branch

Create a new branch using the following command:

git branch feature-login

This creates a new branch named feature-login.

🔄 Switch Branch

Move to another branch using:

git switch feature-login

Older Git versions also support:

git checkout feature-login

📋 View Branches

To see all available branches:

git branch

The current branch is marked with *.

🔀 Merge Branches

After completing your work, merge the branch into the main branch.

git merge feature-login

This combines the changes from feature-login into the current branch.

📸 Expected Output

        $ git branch
        
        * main
          feature-login
          payment-feature

The * symbol indicates the current active branch.

⚠️ Common Mistakes

  • Working directly on the main branch.
  • Forgetting to switch branches.
  • Merging the wrong branch.
  • Deleting a branch before merging.

💡 Pro Tip

Create a separate branch for every new feature or bug fix. It keeps the main branch clean and stable.

🧠 Common Interview Question

Q. Why do developers use branches in Git?

Answer: Branches allow developers to work on different features or bug fixes independently without affecting the main project.

💡 Did You Know?

Large software companies may have hundreds of active branches at the same time for different teams and features.

📋 Key Points

  • git branch creates and lists branches.
  • git switch changes the current branch.
  • git checkout can also switch branches.
  • git merge combines branches.
