Learn how to create a GitHub account, create repositories, and connect your projects with GitHub.

📌 What is GitHub?

GitHub is an online platform that hosts Git repositories. It allows developers to store code, collaborate with teams, and manage projects from anywhere.

💡 Why GitHub?

☁️ Cloud Storage
👥 Team Collaboration
🌍 Access Anywhere
📂 Portfolio Projects

✨ Features of GitHub

📁 Unlimited Repositories
🌿 Branch Management
🐛 Issue Tracking
🚀 GitHub Actions
📖 README Files
🔒 Private Repositories

👤 Create a GitHub Account

  1. Visit https://github.com
  2. Click Sign Up.
  3. Enter your email address.
  4. Create a username and password.
  5. Verify your account.
  6. Complete the registration process.

📂 Create a Repository

  1. Log in to your GitHub account.
  2. Click the New button.
  3. Enter a repository name.
  4. Optionally add a description.
  5. Select Public or Private.
  6. Click Create Repository.

📸 Repository Preview

This is the repository creation page on GitHub. Fill in the repository name and click Create Repository.

💻 Important Commands

Connect your local project with GitHub:

git remote add origin https://github.com/username/repository.git

Upload your project:

git push -u origin main

⚠️ Common Mistakes

  • Creating the repository with the wrong name.
  • Forgetting to add the remote repository.
  • Pushing code without committing changes.
  • Using the wrong branch name.

💡 Pro Tip

Add a meaningful README file to every repository. It helps others understand your project quickly.

🧠 Common Interview Question

Q. What is the difference between Git and GitHub?

Answer: Git is a Version Control System used to track code changes, while GitHub is a cloud platform used to host Git repositories and collaborate with other developers.

💡 Did You Know?

Millions of developers use GitHub every day to contribute to open-source projects and showcase their portfolios.

📋 Key Points

  • GitHub hosts Git repositories online.
  • You need a GitHub account to store repositories.
  • Repositories can be Public or Private.
  • Use git remote add origin to connect a local repository.
  • Use git push to upload code.
