Learn how MySQL stores information using databases, tables, rows, and columns.

Lesson 2 of 10

🗂️ What is a Database?

A database is an organized collection of related data that can be stored, managed, and retrieved efficiently.

It helps applications store information such as student records, customer details, products, and much more.

📋 What is a Table?

A table is a collection of related data arranged in rows and columns.

Each table stores a specific type of information. For example, a Student table stores student details, while a Products table stores product information.

📊 Rows & Columns

📄 Row = One Record
📑 Column = One Field
👨 Student Name
🎂 Student Age

Each row represents one complete record, while each column represents one type of information about that record.

🌍 Real World Example

Imagine a school management system.

The school has one database named SchoolDB. Inside this database, there are multiple tables.

👨‍🎓 Students
👨‍🏫 Teachers
📚 Courses
📝 Results

All these tables together form a complete database for managing the school's information.

💬 Common Interview Question

Question

What is the difference between a Database and a Table?

Answer

A Database is a collection of multiple tables, whereas a Table stores actual data in rows and columns.

💡 Did You Know?

A single MySQL database can store millions of records across multiple tables, making it suitable for both small and large applications.

📝 Key Points

  • ✅ A Database is a collection of related tables.
  • ✅ A Table stores data in rows and columns.
  • ✅ Each row represents one record.
  • ✅ Each column represents one field.
  • ✅ Databases help organize and manage information efficiently.
